# NexForce

NexForce is a cloud gaming frontend concept inspired by premium game-streaming experiences.
The project now runs as a real full-stack app with an Express backend API, persisted app data, and frontend pages wired to live endpoints.

## What this site is for

- Present a cloud gaming brand and product message.
- Showcase a searchable game library and membership plans.
- Run backend-powered demo authentication and profile settings.
- Serve launch estimates (queue, latency, FPS) from API endpoints.

## Tech stack

- HTML5
- Tailwind CSS (CLI build)
- Vanilla JavaScript (ES modules)
- Node.js + Express API
- JSON file persistence in backend storage

## Current backend API

- `GET /api/games`
- `GET /api/plans`
- `POST /api/auth/demo-login`
- `POST /api/auth/logout`
- `GET /api/auth/me`
- `GET /api/profile/settings`
- `PUT /api/profile/settings`
- `PUT /api/profile/plan`
- `GET /api/launch/estimate?plan=free|performance|ultimate`

## What is now real (not mock)

- Games and plans are loaded from backend API.
- Demo auth creates a real server session token.
- Profile settings and selected plan are persisted on backend storage.
- Launch queue/latency/FPS estimates are generated by backend API.

## Run locally

Install dependencies:

```bash
npm install
```

Build CSS:

```bash
npm run build:css
```

Watch CSS while editing:

```bash
npm run watch:css
```

Start local server:

```bash
npm run serve
```

Open:

`http://localhost:5500`

## VPS deployment (no Railway)

This project is now configured for VPS hosting with same-origin API calls (`/api`).

### 1) Server setup

- Install Node.js 20+
- Clone repo and install deps:

```bash
git clone https://github.com/DevEchoX2/NexForce.git
cd NexForce
npm install
npm run build:css
```

### 2) Run with PM2

```bash
npm install -g pm2
pm2 start npm --name nexforce -- run start
pm2 save
pm2 startup
```

### 3) Nginx reverse proxy

Use Nginx to route your domain to Node app on `localhost:5500`:

```nginx
server {
		listen 80;
		server_name wafflev1.me www.wafflev1.me;

		location / {
				proxy_pass http://127.0.0.1:5500;
				proxy_http_version 1.1;
				proxy_set_header Host $host;
				proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
				proxy_set_header X-Forwarded-Proto $scheme;
		}
}
```

Then enable HTTPS:

```bash
sudo certbot --nginx -d wafflev1.me -d www.wafflev1.me
```

### 4) Optional separate API host

If frontend and backend are on different domains, set in a script tag before `app.js`:

```html
<script>
	window.NEXFORCE_API_BASE = "https://api.yourdomain.com/api";
</script>
```

## Notes

This is now a real full-stack web app baseline, but it is still not a real cloud-streaming infrastructure. GPU orchestration, WebRTC media transport, commercial billing, and game entitlement integrations are still future phases.
